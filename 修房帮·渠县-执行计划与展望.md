## 三、注意事项 & 风险提示

### 🚨 技术风险
| 风险 | 影响 | 应对措施 |
|------|------|---------|
| localStorage 数据丢失（清除浏览器缓存） | 用户订单、案例数据全部丢失 | ⚡ **紧急**：尽快迁移至后端数据库 |
| GitHub Pages 部署间歇性不可用 | 页面无法访问（10-30秒重建时间） | 考虑增加 CDN 或迁移至轻量服务器 |
| Server酱 免费额度限制 | 推送失败 | 监控推送量，超过免费额度前升级付费版 |
| 图片存储（微信二维码、用户上传图） | 页面引用失效 | 图片托管至对象存储（OSS/COS），使用永久链接 |

### 🚨 运营风险
| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 虚假订单/恶意提交 | 后台数据污染，推送骚扰 | 增加短信验证码验证，IP限流 |
| 师傅服务质量参差不齐 | 用户差评，平台口碑受损 | 建立实名认证+押金制度 |
| 用户隐私泄露 | 法律风险，信任崩塌 | 手机号等敏感信息加密存储，脱敏显示 |
| 竞争对手模仿 | 市场份额被蚕食 | 快速建立本地师傅网络壁垒+品牌认知 |

### ⚡ 当前最紧急事项（优先处理）
1. **数据持久化**：localStorage → 后端数据库（否则清缓存=全丢）
2. **图片存储**：微信二维码→ OSS/CDN，用户上传图也需要存储方案
3. **HTTPS与安全**：确认 GitHub Pages 强制 HTTPS，防止中间人攻击
4. **短信验证**：用户手机号提交时加验证，防止机器人刷单
5. **隐私合规**：用户协议+隐私政策页面，确认《个人信息保护法》合规

### **第1部分：市场与机会（3页）**

**1.1 县域市场痛点**
- 县城家庭维修/保洁/装修需求旺盛但供给分散
- 传统方式：熟人介绍、路边找师傅、质量无保障
- 「找师傅难、报价不透明、售后无门」三大痛点

**1.2 市场规模**
- 中国县域经济体量：2800+县级行政区，6亿+常住人口
- 家庭服务市场年增速：15%+
- 渠县本地市场估算：年均家庭维修支出2000元/户×20万户≈4亿元

**1.3 我们的优势**
- 更懂本地：渠县本地人运营，了解方言/习惯/小区分布
- 响应更快：30分钟极速上门，本县师傅就近派单
- 信任背书：真实案例+用户评价+售后保障
